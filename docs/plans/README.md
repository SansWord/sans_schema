# Historical tier — plans

Per-milestone **implementation plans** (the "how" agreed after a spec, before
building). **Allowed to go stale, kept forever.** A plan is a snapshot of the
intended steps — *how we meant to build it* — and is **not** the source of truth.

- Don't cite these where they conflict with the code, [`../architecture.md`](../architecture.md),
  or the root `CLAUDE.md`.
- Pair each plan with its spec in [`../specs/`](../specs/); the devlog entry for
  that milestone links both.
- **Kept, never deleted** — the record of how we approached each milestone.

Add new plans alongside this README (e.g. `2026-08-first-gateway.md`).
